# 学习规划：4周掌握实时棋类平台

## 📅 总览

| 阶段 | 时间 | 主题 | 核心目标 |
|------|------|------|----------|
| **第一周** | 第1-7天 | 环境搭建与项目结构 | 成功运行项目，理解整体架构 |
| **第二周** | 第8-14天 | 前端核心技术 | 掌握 React + Zustand + Socket.IO Client |
| **第三周** | 第15-21天 | 后端核心技术 | 掌握 Express + Socket.IO + 游戏引擎 |
| **第四周** | 第22-28天 | 综合实战 | 完成一个功能扩展，融会贯通 |

---

## 🚀 第一周：环境搭建与项目结构

### 目标
- 成功运行前后端项目
- 理解项目整体架构
- 熟悉关键文件位置

### 每日计划

**第1-2天：环境准备**
1. 安装 Node.js（20.x+）
2. 安装 MongoDB（或使用 MongoDB Atlas）
3. 安装 VS Code 及必备插件（ESLint、Prettier）
4. 克隆项目到本地

**第3-4天：运行项目**
1. 进入 `board-game-platform/server` 安装依赖
2. 配置 `.env` 文件（参考现有配置）
3. 启动 MongoDB
4. 启动后端服务（`npm run dev`）
5. 进入 `board-game-platform/client` 安装依赖
6. 启动前端服务（`npm run dev`）
7. 测试注册、登录功能

**第5-6天：熟悉项目结构**
1. 阅读 `docs/系统设计.md` 理解架构
2. 浏览后端目录结构：`server/src/`
3. 浏览前端目录结构：`client/src/`
4. 标记你最感兴趣的3个文件

**第7天：总结与提问**
1. 回顾本周学习内容
2. 列出3个你最困惑的问题
3. 记录项目运行时的现象（如页面跳转、数据流向）

### 关键文件清单
- `server/package.json` - 后端依赖
- `client/package.json` - 前端依赖
- `server/src/index.ts` - 后端入口
- `client/src/App.tsx` - 前端入口

---

## ⚛️ 第二周：前端核心技术

### 目标
- 理解 React 组件结构
- 掌握 Zustand 状态管理
- 理解 Socket.IO 实时通信

### 每日计划

**第8-9天：React 组件与路由**
1. 学习 `client/src/App.tsx` 路由配置
2. 分析 `ProtectedRoute` 组件的作用
3. 查看 `Header.tsx` 组件实现
4. 理解页面组件的职责划分（HomePage、LobbyPage、GamePage）

**第10-11天：Zustand 状态管理**
1. 学习 `client/src/stores/authStore.ts`
2. 理解 login/register/logout 流程
3. 查看 `gameStore.ts` 和 `lobbyStore.ts`
4. 动手练习：在控制台打印状态变化

**第12-13天：Socket.IO 客户端**
1. 学习 `client/src/services/socket.ts`
2. 分析 `useSocket.ts` Hook
3. 追踪 GamePage 中的实时通信
4. 使用浏览器开发者工具查看 WebSocket 消息

**第14天：前端综合练习**
1. 修改 Header 组件显示用户昵称
2. 在 LobbyPage 添加房间数量统计
3. 总结前端数据流

### 关键文件清单
- `client/src/stores/authStore.ts` - 认证状态
- `client/src/services/socket.ts` - Socket 连接
- `client/src/hooks/useSocket.ts` - 自定义 Hook
- `client/src/pages/GamePage.tsx` - 游戏页面

---

## 🖥️ 第三周：后端核心技术

### 目标
- 理解 Express 路由与中间件
- 掌握 Socket.IO 服务端
- 理解游戏引擎设计

### 每日计划

**第15-16天：Express 基础**
1. 学习 `server/src/index.ts` 入口文件
2. 分析路由配置（auth、user、room）
3. 理解 JWT 中间件 `middleware/auth.ts`
4. 学习错误处理中间件

**第17-18天：Socket.IO 服务端**
1. 学习 `server/src/socket/index.ts`
2. 分析认证中间件
3. 查看 `roomHandler.ts` 和 `gameHandler.ts`
4. 理解房间机制和事件处理

**第19-20天：游戏引擎**
1. 学习 `server/src/game-engine/BaseGame.ts`
2. 深入分析 `ChineseChess.ts` 中的走法验证
3. 理解 `validateMove` 和 `makeMove` 的流程
4. 查看 `types.ts` 类型定义

**第21天：数据库与业务逻辑**
1. 学习 Mongoose 模型（User、Room、GameRecord）
2. 分析 `services/` 目录下的业务逻辑
3. 理解数据流向：API → Service → Model

### 关键文件清单
- `server/src/socket/index.ts` - Socket 初始化
- `server/src/game-engine/BaseGame.ts` - 游戏基类
- `server/src/game-engine/ChineseChess.ts` - 象棋引擎
- `server/src/middleware/auth.ts` - JWT 中间件

---

## 🎯 第四周：综合实战

### 目标
- 完成一个功能扩展
- 融会贯通前后端知识
- 建立完整的项目理解

### 每日计划

**第22-23天：选择功能并设计**
1. 从以下选项中选择一个功能：
   - ✅ 添加游戏计时器
   - ✅ 实现悔棋功能
   - ✅ 添加房间密码保护
   - ✅ 实现聊天功能完善
2. 设计前后端接口和事件
3. 绘制流程图

**第24-26天：编码实现**
1. 实现后端逻辑（路由/Socket事件/游戏引擎）
2. 实现前端组件和状态管理
3. 测试功能是否正常工作

**第27-28天：测试与总结**
1. 完整测试新功能
2. 修复 Bug
3. 总结所学知识
4. 记录学习心得

### 推荐实战项目

#### 项目1：游戏计时器
- 后端：在房间中添加计时逻辑
- Socket 事件：`timer_tick`, `timer_end`
- 前端：显示倒计时，超时自动判负

#### 项目2：悔棋功能
- 后端：保存历史状态，支持回退
- Socket 事件：`undo_move`
- 前端：添加悔棋按钮

## 💡 学习技巧

1. **断点调试**：在关键代码处设置断点，观察数据流向
2. **日志输出**：在服务端和客户端添加 console.log
3. **画图理解**：用思维导图绘制架构和流程
4. **最小改动**：每次只改一小部分，测试通过再继续
5. **提问记录**：遇到问题先记录，积累到一定数量再集中解决
