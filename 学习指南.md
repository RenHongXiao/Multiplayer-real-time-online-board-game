# 实时在线多人棋类平台学习指南

## 项目概述

这是一个基于 **React + Node.js + Socket.IO** 构建的实时在线多人棋类游戏平台，支持中国象棋和五子棋两种棋类游戏。

## 🚀 快速开始

### 环境要求
- Node.js >= 20.x
- MongoDB >= 8.x
- npm >= 10.x

### 启动项目

```bash
# 进入项目目录
cd board-game-platform

# 安装后端依赖
cd server
npm install

# 安装前端依赖
cd ../client
npm install

# 启动服务（确保 MongoDB 已运行）
# 方式1：使用一键启动脚本
../一键启动.bat

# 方式2：手动启动
# 后端
cd server && npm run dev
# 前端
cd client && npm run dev
```

## 🔐 认证机制

### JWT 认证流程

```
1. 用户登录 → 服务端验证 → 返回 JWT Token
2. Token 存储在 localStorage
3. 后续请求通过 Authorization: Bearer <token> 携带
4. WebSocket 连接时通过 handshake.auth.token 传递
```

### 核心代码示例

**后端认证中间件** (`server/src/middleware/auth.ts`)：

```typescript
// JWT 验证逻辑
const decoded = jwt.verify(token, config.jwtSecret) as { userId: string };
socket.userId = decoded.userId;
```

**前端状态管理** (`client/src/stores/authStore.ts`)：

```typescript
login: async (username, password) => {
  const { data } = await api.post('/auth/login', { username, password });
  localStorage.setItem('token', token);
  connectSocket(token); // 登录成功后连接 Socket
}
```

## 🔄 实时通信机制

### Socket.IO 事件架构

| 事件类型 | 事件名 | 说明 |
|---------|--------|------|
| 房间事件 | `create_room` | 创建房间 |
| | `join_room` | 加入房间 |
| | `leave_room` | 离开房间 |
| | `list_rooms` | 获取房间列表 |
| 游戏事件 | `make_move` | 落子 |
| | `game_state` | 游戏状态同步 |
| | `opponent_disconnect` | 对手断开 |

### 走棋流程

```
客户端点击棋盘
    ↓
验证合法性（本地初步校验）
    ↓
发送 Socket 事件 `make_move`
    ↓
服务端验证走法（核心校验）
    ↓
更新游戏状态
    ↓
广播 `game_state` 给房间内所有玩家
    ↓
客户端接收状态并渲染
```

## 🎮 游戏引擎设计

### 抽象基类设计

**BaseGame** (`server/src/game-engine/BaseGame.ts`) 定义了游戏引擎的核心接口：

```typescript
abstract class BaseGame {
  protected board: any[][];           // 棋盘状态
  protected currentTurn: string;       // 当前回合
  protected moves: Move[];             // 走棋记录
  protected isOver: boolean;           // 游戏是否结束
  protected result: GameResult | null; // 游戏结果

  abstract initialize();               // 初始化棋盘
  abstract validateMove();             // 验证走法合法性
  abstract makeMove();                 // 执行走棋
  abstract checkGameOver();            // 检查游戏结束
  abstract getLegalMoves();            // 获取合法走法
}
```

### 中国象棋引擎

**ChineseChess** (`server/src/game-engine/ChineseChess.ts`) 继承自 BaseGame，实现了：

- **棋子布局**：10×9 棋盘，双方各 16 枚棋子
- **走法规则**：车、马、炮、象、士、将、兵的独特走法
- **特殊规则**：将帅不能直接对面（飞将规则）
- **胜负判定**：将死对方将帅获胜

### 关键实现细节

```typescript
// 走法验证流程
validateMove(fromX, fromY, toX, toY) {
  1. 边界检查 → 2. 棋子归属检查 → 3. 走法规则检查 → 4. 将军检查
}

// 将军检测
isKingInCheck(color) {
  // 1. 飞将规则检查
  // 2. 检查对方所有棋子是否能攻击己方将帅
}
```

## 🐛 常见问题

### 1. Socket 连接失败
- 检查服务端是否启动
- 检查 Token 是否有效
- 检查 CORS 配置

### 2. 走棋不生效
- 检查服务端走法验证逻辑
- 检查客户端状态更新
- 检查 Socket 事件监听

### 3. 棋盘渲染异常
- 检查 Canvas 绑定的 state
- 检查游戏状态同步时机
