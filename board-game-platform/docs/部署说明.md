# 实时在线多人棋类平台 —— 部署说明

## 1. 环境要求

| 软件 | 版本 | 说明 |
|------|------|------|
| Node.js | 18+ | JavaScript 运行时 |
| npm | 9+ | 包管理器 |
| MongoDB | 6+ | 数据库（本地或 Atlas 云） |

## 2. 安装步骤

### 2.1 克隆项目
```bash
cd board-game-platform
```

### 2.2 安装后端依赖
```bash
cd server
npm install
```

### 2.3 安装前端依赖
```bash
cd ../client
npm install
```

### 2.4 配置环境变量

编辑 `server/.env`：
```env
PORT=3000
MONGO_URI=mongodb://localhost:27017/board-game
JWT_SECRET=your-secret-key-here
JWT_EXPIRES_IN=7d
CORS_ORIGIN=http://localhost:5173
```

如果使用 MongoDB Atlas 云服务，将 MONGO_URI 改为 Atlas 连接字符串：
```
MONGO_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/board-game
```

### 2.5 启动 MongoDB
```bash
# 本地 MongoDB
mongod --dbpath <你的数据目录路径>
```

## 3. 运行项目

### 3.1 开发模式

启动后端（端口 3000）：
```bash
cd server
npm run dev
```

启动前端（端口 5173）：
```bash
cd client
npm run dev
```

访问：`http://localhost:5173`

### 3.2 生产构建

构建后端：
```bash
cd server
npm run build
npm start
```

构建前端：
```bash
cd client
npm run build
```
前端构建产物在 `client/dist/`，可使用 Nginx 或 Express 静态托管。

## 5. 常见问题

**Q: MongoDB 连接失败？**  
A: 检查 MongoDB 服务是否启动，或 MONGO_URI 配置是否正确。

**Q: 前端请求后端跨域？**  
A: 开发环境下 Vite 已配置代理，确保 Vite 开发服务器在 5173 端口运行。

**Q: WebSocket 连接失败？**  
A: 确认用户已登录（有 JWT Token），Socket 连接需要认证。

**Q: 需要禁用 PowerShell 执行策略？**  
A: 以管理员身份运行 `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`
