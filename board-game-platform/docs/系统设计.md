## 2. 技术选型说明

### 2.1 后端技术

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| Node.js | 24.x | 高性能异步 I/O，适合实时应用 |
| TypeScript | 5.6 | 类型安全，工程化程度高 |
| Express | 4.21 | 成熟稳定的 Web 框架，生态丰富 |
| Socket.IO | 4.8 | 自动降级、房间机制、断线重连，比原生 WebSocket 更稳定 |
| MongoDB | 8.x | 文档型数据库，灵活的数据结构适合游戏场景 |
| Mongoose | 8.7 | MongoDB ODM，提供 Schema 校验和关系映射 |
| JWT | 9.0 | 无状态认证，适合分布式部署 |
| bcryptjs | 2.4 | 密码哈希，防彩虹表 |
| Zod | 3.23 | TypeScript-first 校验，类型推导 |

### 2.2 前端技术

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| React | 18 | 组件化 UI 库，生态庞大 |
| TypeScript | 5.6 | 类型安全 |
| Vite | 5.x | 极速开发服务器，HMR |
| React Router | 6.x | 声明式路由 |
| Zustand | 5.x | 轻量状态管理，比 Redux 更简洁 |
| Canvas API | - | 高性能棋盘渲染，支持细腻绘制 |
| Socket.IO Client | 4.8 | 实时通信，自动重连 |
| TailwindCSS | 4.x | 原子化样式，快速开发 |
| Axios | 1.x | HTTP 请求库，拦截器机制 |

## 4. 安全设计

- **密码加密**：bcrypt 盐值哈希（saltRounds=10）
- **JWT 认证**：REST API 通过 Bearer Token 鉴权，WebSocket 连接时传递 Token
- **走法校验**：所有走棋操作在服务端验证，防止客户端篡改
- **输入校验**：Zod Schema 校验所有 API 输入
- **SQL 注入防护**：使用 Mongoose ORM，参数化查询
- **CORS 限制**：仅允许前端域名跨域访问
