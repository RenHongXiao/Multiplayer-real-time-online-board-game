#### 中国象棋引擎架构 `game-engine/ChineseChess.ts`

关键方法调用链：

```
validateMove(from, to)
  ├── isInBounds()              → 边界检查
  ├── piece.color === currentTurn? → 回合检查（skipTurnCheck 可跳过）
  ├── pieceCanMove()            → 棋子走法规则
  │   ├── kingCanMove()         → 将/帅：九宫内一步
  │   ├── advisorCanMove()      → 士/仕：九宫内斜一步
  │   ├── elephantCanMove()     → 象/相：田字 + 塞眼检测
  │   ├── knightCanMove()       → 马：日字 + 蹩脚检测
  │   ├── rookCanMove()         → 车：直线 + 阻挡检测
  │   ├── cannonCanMove()       → 炮：直线（走）/跳一吃（吃）
  │   └── pawnCanMove()         → 兵/卒：前进/过河左右
  └── isKingInCheck()           → 走后不将军？（模拟+回滚）

makeMove(from, to)
  ├── validateMove()            → 合法性校验
  ├── 执行移动 + 记录棋谱
  ├── checkGameOver()           → 对手是否无路可走？
  │   ├── hasAnyLegalMove()     → 遍历对手所有棋子
  │   └── isKingInCheck()       → 判断绝杀/困毙
  └── 切换回合（如果游戏未结束）
```

```

  Header: Authorization: Bearer <JWT_TOKEN>
  中间件提取 → JWT 验证 → req.userId

WebSocket 认证：
  连接时：socket.handshake.auth.token = "<JWT_TOKEN>"
  io.use() 中间件 → JWT 验证 → socket.userId

JWT 结构：
  Header: { alg: "HS256", typ: "JWT" }
  Payload: { userId: "<mongo_id>", iat: ..., exp: ... }
  Signature: HMAC-SHA256(payload, jwtSecret)
```

### 7.2 各棋子走法规则

| 棋子 | 走法 | 特殊限制 |
|------|------|----------|
| 将/帅 | 直线一步 | 限九宫（3×3） |
| 士/仕 | 斜线一步 | 限九宫 |
| 象/相 | 田字对角 | 不能过河 + 塞象眼 |
| 马 | 日字 | 蹩脚检测 |
| 车 | 直线任意 | 不可越子 |
| 炮 | 直线（走）/ 跳一吃（吃） | 走时不可越子，吃时必须跳一个 |
| 兵/卒 | 前进一步 / 过河可左右 | 不可后退 |

### 7.3 将军与绝杀判定

```
将军检测 (isKingInCheck)：
  1. 飞将检测：两将在同列无阻挡 → 将军
  2. 遍历对方所有棋子 → pieceCanMove 是否可达己方将位置

绝杀 vs 困毙 (checkGameOver)：
  遍历对方所有棋子的所有合法走法
  ├── 有合法走法 → 游戏继续
  └── 无合法走法
      ├── 被将军 → 绝杀 (checkmate)
      └── 未被将 → 困毙 (stalemate)
```

### 7.4 五子棋胜负判定

```
每次落子后，检查四个方向：
  → (0,1)  横
  ↓ (1,0)  竖
  ↘ (1,1)  斜
  ↙ (1,-1) 反斜

每个方向正反各数 4 步，连续同色数 ≥ 5 即获胜
```

## 8. 实时通信架构

### 8.1 Socket.IO 房间机制

```
房间生命周期：
  用户A 创建房间 → MongoDB 保存 → Socket.join(roomId) → 广播 room_update
  用户B 加入房间 → MongoDB 更新 → Socket.join(roomId)
  双方 ready   → 创建 GameSession → 双方 game_start
  走棋循环     → make_move → 服务端验证 → 广播 move_made
  游戏结束     → 保存记录 → 更新战绩 → 广播 game_over → 关闭房间
```

### 8.2 GameSession 内存结构

```typescript
// 存储在服务端内存中的游戏会话（不持久化）
interface GameSession {
  roomId: string;
  gameType: 'chinese-chess' | 'gomoku';
  players: Map<socketId, color>;     // 玩家连接映射
  gameInstance: ChineseChess | Gomoku; // 游戏引擎实例
  recordId: string;                   // MongoDB 对局记录 ID
  timer: {
    red: number;      // 红方剩余秒数（默认 1200）
    black: number;    // 黑方剩余秒数
    lastTick: number; // 上次计时时间戳
    interval: NodeJS.Timeout; // 每秒更新
  };
}
```

### 8.3 断线重连

```
Socket.IO 配置：
  reconnection: true
  reconnectionAttempts: 10
  reconnectionDelay: 1000ms
  reconnectionDelayMax: 5000ms

断线处理：
  socket.on('disconnect') → 遍历该用户所在房间
  → socket.to(room).emit('opponent_disconnect', { userId, timestamp })
  → 前端显示"对手已断线"提示
```

## 9. Bug 修复记录

### Bug #1：初始局面误判将军

**现象**：中国象棋引擎初始化后，`isKingInCheck('red')` 返回 `true`，导致所有红方走法都不可用。

**根因**：`pieceCanMoveDirectly()` 中对 `king` 类型的处理过于宽松，只检查 `isInBounds`（目标是否在棋盘内），导致黑将能从 (0,4) "攻击"红帅 (9,4)，实际上九宫内的将/帅不可能跨 9 行攻击。

**文件**：[ChineseChess.ts:306-309](file:///d:/Comprehensive%20Computer%20Design1/board-game-platform/server/src/game-engine/ChineseChess.ts#L306-L309)

**修复前**：
```typescript
if (piece.type === 'king') {
  return this.isInBounds(tx, ty, this.ROWS, this.COLS);
}
```

**修复后**：飞将规则已在 `isKingInCheck` 中单独处理（同列无阻挡），所以在循环遍历对方棋子时应排除对方将/帅：
```typescript
if (piece.type === 'king') return false;
```

### Bug #2：走棋后被误判困毙

**现象**：红方走炮 (7,1)→(7,4) 后，`checkGameOver()` 判定黑方无合法走法且未被将军 → 困毙 → 红方胜。实际上黑方刚开局应有大量可走。

**根因**：`checkGameOver()` 调用 `hasAnyLegalMove('black')` → `getLegalMoves()` → `validateMove()`，而 `validateMove()` 中有回合检查 `piece.color !== this.currentTurn`。此时 `currentTurn` 仍是红方（回合尚未切换），导致所有黑方走法都被回合检查拦截。

**修复涉及 3 个文件**：

1. `BaseGame.ts` — 抽象方法签名增加 `skipTurnCheck` 参数
2. `ChineseChess.ts` — `validateMove` 和 `getLegalMoves` 增加可选参数
3. `Gomoku.ts` — 方法签名同步更新

**关键修改** (`ChineseChess.ts`):
```typescript
// validateMove 增加可选参数
validateMove(fromX, fromY, toX, toY, skipTurnCheck = false): boolean {
  // ...
  if (!skipTurnCheck && piece.color !== this.currentTurn) return false;
  // ...
}

// getLegalMoves 传递 skipTurnCheck
getLegalMoves(x, y, skipTurnCheck = false): ... {
  if (this.validateMove(x, y, toX, toY, skipTurnCheck)) { ... }
}

// checkGameOver 中跳过回合检查
private hasAnyLegalMove(color): boolean {
  const moves = this.getLegalMoves(x, y, true); // ← skipTurnCheck=true
}
```
