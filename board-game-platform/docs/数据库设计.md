# 实时在线多人棋类平台 —— 数据库设计

## 1. 数据库选择

采用 **MongoDB** 文档数据库，优势：
- 面向文档，数据结构灵活，适合游戏数据的多变需求
- 支持嵌套文档，走棋记录可直接内嵌在 GameRecord 中
- Mongoose ODM 提供 Schema 校验和 Population（联表查询）

## 2. 集合设计

### 2.1 User（用户）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| username | String | 用户名，唯一索引，3-20字符 |
| passwordHash | String | bcrypt 加密后的密码 |
| nickname | String | 显示昵称 |
| avatar | String | 头像 URL，默认空 |
| stats.totalGames | Number | 总对局数，默认 0 |
| stats.wins | Number | 胜利数，默认 0 |
| stats.losses | Number | 失败数，默认 0 |
| stats.draws | Number | 和棋数，默认 0 |
| stats.rating | Number | 天梯分，默认 1000 |
| createdAt | Date | 创建时间（自动） |
| updatedAt | Date | 更新时间（自动） |

### 2.2 Room（房间）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| roomId | String | 6位房间号，唯一索引 |
| gameType | String | 游戏类型：'chinese-chess' / 'gomoku' |
| hostId | ObjectId | 房主用户 ID（ref: User） |
| player2Id | ObjectId | 加入者 ID，可为 null（ref: User） |
| status | String | 状态：'waiting' / 'playing' / 'finished' |
| maxPlayers | Number | 最大玩家数，默认 2 |
| createdAt | Date | 创建时间（自动） |

### 2.3 GameRecord（对局记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| roomId | String | 关联房间号 |
| gameType | String | 游戏类型 |
| players | Array | 玩家信息 |
| players[].userId | ObjectId | 用户 ID（ref: User） |
| players[].color | String | 颜色：'red' / 'black' / 'white' |
| moves | Array | 走棋记录 |
| moves[].from | {x,y} | 起始位置 |
| moves[].to | {x,y} | 目标位置 |
| moves[].piece | String | 棋子标识（如 red_rook） |
| moves[].timestamp | Date | 走棋时间 |
| result.winner | ObjectId? | 胜方 ID，null=和棋（ref: User） |
| result.reason | String | 结束原因 |
| startedAt | Date | 开始时间 |
| endedAt | Date | 结束时间 |

## 3. 结束原因枚举

| 值 | 说明 |
|------|------|
| checkmate | 绝杀（中国象棋将死） |
| stalemate | 困毙（无子可走） |
| resign | 主动认输 |
| timeout | 超时判负 |
| five-in-row | 五子连珠（五子棋） |
| draw | 协议和棋 |
